# Report Format

## Final API Report Structure

The API audit always follows a fixed structure:

1. API аудит процесса
2. Резюме
3. Автоматизация (Уровень 1–3)
4. Быстрые улучшения
5. ROI гипотезы
6. Ограничения

## Formatting rules

- short headings
- no parentheses in titles
- unified writing style
- no model questions
- no references like “based on card X”
- no industry fantasy assumptions

## Normalization

Before final output:

- report is normalized
- formatting is validated
- broken outputs trigger retry
